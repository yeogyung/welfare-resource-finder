# IA 및 구현 범위

## IA 출처

- `찾아봇 AI 기획서_수정_최종반영.xlsx`
- Figma `찾아봇` 디자인 화면
- Figma HTML 추출 코드
- 챗봇 상태 이미지: 기본, 궁금, 듣는 중, 손인사

## 구현 화면

| 화면 | 구현 내용 |
|---|---|
| 홈 | 새로고침마다 달라지는 첫 질문, 생활 도움/활동·여가 빠른 질문, 음성 입력, 직접 입력 |
| 대화 | 사용자 말풍선, 찾아봇 말풍선, 후속 질문, 추천 카드 |
| 탐색 | 전체 복지자원 검색, 카테고리 필터, DB 출처 확인 |
| 찜 | 저장한 자원 목록, 생활 도움/활동·여가 축 필터, 전화/상세/찜 취소 |

## 내부 운영 화면

| 화면 | 구현 내용 |
|---|---|
| 관리자 평가 | 교수 피드백 기반 RAG/검색 예비 평가 실행, 평가 결과 JSON 다운로드 |

사용자 프로토타입에는 RAG 평가, 모델 점수, Faithfulness 등 내부 지표를 노출하지 않는다. 해당 기능은 `/admin.html`에서 내부 관리자/개발자가 확인한다.

## 핵심 사용자 흐름

1. 어르신이 앱에 진입한다.
2. 찾아봇이 랜덤 첫 질문으로 생활 불편, 건강, 경제·주거, 배움, 문화·여가, 고립감 등 다양한 상황을 열어 둔다.
3. 어르신이 음성, 텍스트, 빠른 질문 중 편한 방식으로 현재 상황을 표현한다.
4. 찾아봇이 질문 의도를 `생활 도움` 또는 `활동·여가` 축과 세부 카테고리로 분류한다.
5. 복지자원 DB에서 근거 자원을 검색하고, 추천 카드에 대상, 지역, 신청방법, 출처, URL을 표시한다.
6. 필요한 자원은 찜에 저장하고, 찜 화면에서 두 축 기준으로 다시 확인한다.
7. 응급/위기 질문은 일반 추천보다 안전 안내를 먼저 제공한다.

## 기획 축

찾아봇은 특정 걱정만 묻는 상담형 서비스가 아니라, 대화를 통해 사용자의 상황을 발견하고 복지자원 탐색으로 연결하는 모바일 웹앱이다. 따라서 홈의 첫 질문은 고정 문장이 아니라 새로고침 또는 재진입 시 랜덤하게 바뀌며, 빠른 질문은 다음 두 축으로 구성한다.

| 축 | 사용자 상황 | 예시 질문 |
|---|---|---|
| 생활 도움 | 돌봄, 건강, 경제, 주거, 응급·안전 | 생활에 도움이 필요해요 / 병원·건강이 궁금해요 / 생활비나 집이 걱정돼요 |
| 활동·여가 | 배움, 문화, 모임, 사회참여, 고립감 완화 | 뭔가 배우고 싶어요 / 공연이나 모임을 찾고 싶어요 / 혼자 있는 시간이 많아요 |

이 구조는 사용자가 처음부터 정확한 복지서비스명을 알지 못해도 생활 맥락을 말하게 하고, 내부적으로는 DB 카테고리와 검색 로직에 연결되도록 설계한 것이다.

## 디자인 반영 사항

- Figma 기준 375px 모바일 화면을 중심으로 구현
- 배경색 `#F3F4F6`, 메인 브랜드 색상 `#3A8F7A`
- 8px 카드 radius, 16px 말풍선 radius
- Pretendard/SF Pro 계열 타이포그래피
- 고령층 접근성을 위해 기본 글자 크기를 크게 설정
- 하단 고정 내비게이션 구성
- 캐릭터 상태 이미지로 대화 상태 표현
- 사용자 앱 하단 내비게이션은 홈/탐색/찜으로 제한
- 홈 빠른 질문과 찜 필터에 `생활 도움`/`활동·여가` 축 반영

## 개발 산출물 포인트

- IA/와이어프레임을 실제 웹앱 화면 구조로 전환
- Figma 시안을 정적 HTML/CSS/JS로 구현
- 복지자원 엑셀을 앱에서 검색 가능한 JSON 데이터로 변환
- 랜덤 첫 질문과 두 축 기반 빠른 질문을 구현
- STT/TTS 기반 음성 접근성 구현
- RAG 평가 설계를 사용자 앱과 분리된 내부 관리자 화면으로 구현
