# IA 및 구현 범위

## IA 출처

- `찾아봇 AI 기획서_수정_최종반영.xlsx`
- Figma `찾아봇` 디자인 화면
- Figma HTML 추출 코드
- 확정 프로토타입 HTML `index_2_확정프로토타입.html`
- 챗봇 상태 이미지: 기본, 궁금, 듣는 중, 손인사

## 구현 화면

| 화면 | 구현 내용 |
|---|---|
| 홈 | 새로고침마다 달라지는 첫 질문 7종, 랜덤 입력 예시 6종, 음성 입력, 직접 입력, 빠른 진입 메뉴 6종 |
| 대화 | 랜덤 첫 질문 3종, 꼬리질문 트리, 사용자 말풍선, 찾아봇 말풍선, 추천 카드 |
| 찜 | 저장한 자원 목록, 생활형/여가·문화·배움 축 확인, 전화/상세/찜 취소, 공유 |
| 설정 | 큰 글자, 음성 안내, 지역 설정, 보호자·활동가 공유 |

## 내부 운영 화면

| 화면 | 구현 내용 |
|---|---|
| 관리자 평가 | 교수 피드백 기반 RAG/검색 예비 평가 실행, 평가 결과 JSON 다운로드 |

사용자 프로토타입에는 RAG 평가, 모델 점수, Faithfulness 등 내부 지표를 노출하지 않는다. 해당 기능은 `/admin.html`에서 내부 관리자/개발자가 확인한다.

## 핵심 사용자 흐름

1. 어르신이 앱에 진입한다.
2. 찾아봇이 랜덤 첫 질문으로 필요(생활형)와 욕구(여가·문화·배움)를 번갈아 열어 둔다.
3. 어르신이 음성, 텍스트, 빠른 질문 중 편한 방식으로 현재 상황을 표현한다.
4. 찾아봇이 질문 의도를 `필요(생활형)` 또는 `욕구(여가·문화·배움)` 축과 세부 카테고리로 분류한다.
5. 복지자원 DB에서 근거 자원을 검색하고, 추천 카드에 대상, 지역, 신청방법, 출처, URL을 표시한다.
6. 필요한 자원은 찜에 저장하고, 설정에서 보호자·활동가 공유를 실행할 수 있다.
7. 응급/위기 질문은 일반 추천보다 안전 안내를 먼저 제공한다.

## 기획 축

찾아봇은 특정 걱정만 묻는 상담형 서비스가 아니라, 대화를 통해 사용자의 상황을 발견하고 복지자원 탐색으로 연결하는 모바일 웹앱이다. 발견 방식은 두 축으로 설계한다. 하나는 끼니·돌봄·건강·생계·안전처럼 결핍을 해결하는 `필요(생활형)`이고, 다른 하나는 모임·취미·나들이·디지털 배움처럼 삶을 넓히는 `욕구(여가·문화·배움)`이다.

기존 복지 포털은 사용자가 `기초연금`처럼 정확한 서비스명을 알아야 검색이 가능하지만, 찾아봇은 매번 다른 질문을 먼저 던져 두 축을 번갈아 노출한다. 사용자는 대화 중에 `내가 받을 수 있는 게 이런 것도 있구나`를 발견하게 되며, 이 점에서 검색이 아니라 대화가 발견의 경로가 된다.

| 축 | 사용자 상황 | 예시 질문 |
|---|---|---|
| 필요(생활형) | 끼니, 돌봄, 건강, 생계, 안전, 주거 | 끼니·생활이 걱정돼요 / 건강이 걱정돼요 / 생활비가 빠듯해요 |
| 욕구(여가·문화·배움) | 경로당, 모임, 취미, 나들이, 문화행사, 디지털 배움 | 즐길 거리·모임을 찾고 싶어요 / 새로운 걸 배우고 싶어요 |

이 구조는 사용자가 처음부터 정확한 복지서비스명을 알지 못해도 생활 맥락을 말하게 하고, 내부적으로는 DB 카테고리와 검색 로직에 연결되도록 설계한 것이다.

## 확정 질문 구조

### 홈 랜덤 질문 7종

- 오늘은 무엇을 찾아드릴까요?
- 요즘 어떤 점이 걱정되세요?
- 지금 가장 필요한 게 있으세요?
- 요즘 해보고 싶은 게 있으세요?
- 어떤 걸 배우거나 즐기고 싶으세요?
- 혼자 지내시기 불편한 점이 있으세요?
- 오늘은 어떤 도움을 함께 찾아볼까요?

### 입력 예시 랜덤 6종

- 예: 혼자 살아서 끼니가 걱정돼요
- 예: 스마트폰 쓰는 법을 배우고 싶어요
- 예: 경로당 모임에 나가고 싶어요
- 예: 병원 가기가 힘들어요
- 예: 생활비가 부족해요
- 예: 가까운 곳에서 즐길 거리가 있을까요

### 대화 첫 질문 랜덤 3종

- 무엇을 도와드릴까요? 필요한 것도, 해보고 싶은 것도 편하게 골라주세요.
- 어떤 이야기든 좋아요. 어떤 점이 궁금하세요?
- 필요한 복지든 즐길 거리든, 어떤 걸 찾아드릴까요?

### 빠른 진입 메뉴 6종

| 축 | 메뉴 |
|---|---|
| 필요(생활형) | 끼니·생활이 걱정돼요 / 외롭고 말벗이 필요해요 / 건강이 걱정돼요 / 생활비가 빠듯해요 |
| 욕구(여가·문화·배움) | 즐길 거리·모임을 찾고 싶어요 / 새로운 걸 배우고 싶어요 |

### 여가·문화 분기

`즐길 거리·모임을 찾고 싶어요`를 선택하면 다음 세 가지로 이어진다.

- 경로당·모임에 나가고 싶어요
- 취미·여가 프로그램을 배우고 싶어요
- 나들이·문화 행사에 가보고 싶어요

## 디자인 반영 사항

- Figma 기준 375px 모바일 화면을 중심으로 구현
- 배경색 `#F3F4F6`, 메인 브랜드 색상 `#3A8F7A`
- 8px 카드 radius, 16px 말풍선 radius
- Pretendard/SF Pro 계열 타이포그래피
- 고령층 접근성을 위해 기본 글자 크기를 크게 설정
- 하단 고정 내비게이션 구성
- 캐릭터 상태 이미지로 대화 상태 표현
- 사용자 앱 하단 내비게이션은 홈/찜/설정으로 제한
- 홈 빠른 질문과 추천 카드에 `필요(생활형)`/`욕구(여가·문화·배움)` 축 반영

## 개발 산출물 포인트

- IA/와이어프레임을 실제 웹앱 화면 구조로 전환
- Figma 시안을 정적 HTML/CSS/JS로 구현
- 복지자원 엑셀을 앱에서 검색 가능한 JSON 데이터로 변환
- 랜덤 홈 질문, 랜덤 입력 예시, 랜덤 대화 첫 질문을 구현
- 필요(생활형)과 욕구(여가·문화·배움) 두 축 기반 대화 트리 구현
- STT/TTS 기반 음성 접근성 구현
- RAG 평가 설계를 사용자 앱과 분리된 내부 관리자 화면으로 구현
