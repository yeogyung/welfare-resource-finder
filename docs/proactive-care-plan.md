# 찾아봇 상황 발견형 질문 UX 설계

## 기획 의도

찾아봇은 특정 걱정만 묻는 상담형 챗봇이 아니라, 어르신이 복지서비스명을 몰라도 생활 맥락을 말할 수 있게 돕는 복지자원 탐색 인터페이스다. 홈 화면의 첫 질문은 고정 문장이 아니라 랜덤으로 바뀌며, 질문 범위는 생활 도움과 활동·여가를 모두 포함한다.

## MVP 반영

- 홈 화면 첫 질문을 새로고침 또는 재진입 시 랜덤으로 노출한다.
- 빠른 질문을 `생활 도움`과 `활동·여가` 두 축으로 나눈다.
- `생활 도움`은 돌봄, 건강, 경제, 주거, 응급·안전 자원으로 연결한다.
- `활동·여가`는 배움, 문화, 모임, 사회참여, 고립감 완화 자원으로 연결한다.
- `질문 듣기` 버튼은 현재 홈 질문을 포함한 안내 문장을 음성으로 읽는다.
- 브라우저 자동재생 정책 때문에 음성 재생은 사용자 버튼 입력 이후 시작하도록 설계했다.

## 랜덤 첫 질문 후보

- 오늘 어떤 도움이 필요하세요?
- 요즘 생활하면서 불편한 게 있으세요?
- 어디 가서 뭔가 해보고 싶은 마음이 있으세요?
- 건강이나 병원 때문에 신경 쓰이는 게 있으세요?
- 혼자 있는 시간이 길게 느껴지세요?
- 생활비나 집 문제로 신경 쓰이는 게 있으세요?
- 새로 배우거나 참여해 보고 싶은 활동이 있으세요?

## 두 축 분류

| 축 | 포함 범위 | 대표 UI 문장 |
|---|---|---|
| 생활 도움 | 돌봄, 건강·의료, 경제, 주거, 응급·안전 | 생활에 도움이 필요해요 / 병원·건강이 궁금해요 / 생활비나 집이 걱정돼요 |
| 활동·여가 | 배움, 문화, 모임, 사회참여, 고립감 완화 | 뭔가 배우고 싶어요 / 공연이나 모임을 찾고 싶어요 / 혼자 있는 시간이 많아요 |

## 대화 흐름

1. 홈에서 랜덤 첫 질문과 두 축 빠른 질문을 보여준다.
2. 사용자는 음성, 텍스트, 빠른 질문 중 편한 방식으로 상황을 표현한다.
3. 찾아봇은 질문을 축과 세부 카테고리로 분류한다.
4. DB 검색 결과를 근거 카드로 제시한다.
5. 사용자는 필요한 자원을 찜에 저장하고, 찜 화면에서 `생활 도움`/`활동·여가`로 다시 확인한다.
6. 응급·안전 표현이 감지되면 일반 추천보다 안전 안내와 공식 자원을 우선한다.

## 음성 기술 선택

### 현재 MVP

- STT: 브라우저 Web Speech API
- TTS: 브라우저 `speechSynthesis`
- 장점: 비용 없이 즉시 배포 가능
- 한계: 브라우저/기기별 음성 품질과 지원 범위 차이

### 운영 버전

- STT: NAVER Cloud CLOVA Speech
- TTS: NAVER Cloud CLOVA Voice
- API 키는 프론트엔드에 넣지 않고 Vercel Serverless Function 또는 별도 백엔드에 보관한다.
- 한국어 음성 품질과 운영 안정성을 고려하면 실증용 운영 스택으로 적합하다.

## 결과보고서 표현

> 찾아봇은 단순 검색창이 아니라 사용자가 복지서비스명을 몰라도 생활 맥락을 말할 수 있도록 돕는 상황 발견형 대화 인터페이스로 설계하였다. MVP에서는 홈 첫 질문을 랜덤화하고, 빠른 질문을 생활 도움/활동·여가 두 축으로 구성했으며, 브라우저 내장 STT/TTS로 음성 질문과 음성 안내를 구현하였다. 실증 버전에서는 NAVER Cloud CLOVA Speech/Voice를 서버리스 API로 연동하여 한국어 음성 품질과 운영 안정성을 높이는 구조를 제안하였다.
