# 찾아봇 선제 안부 대화 설계

## 기획 의도

찾아봇은 사용자가 질문을 입력하기만 기다리는 검색창이 아니라, 어르신에게 먼저 짧게 안부를 묻고 필요한 복지자원을 함께 찾아주는 대화형 안내자다.

## MVP 반영

- 홈 화면에서 찾아봇이 먼저 "안녕하세요, 찾아봇이에요"라고 인사하는 구조를 배치했다.
- `인사 듣기` 버튼을 누르면 찾아봇이 먼저 안부 질문을 음성으로 읽는다.
- `음성 안내 켜기`를 선택하면 이후 찾아봇 말풍선도 음성으로 안내한다.
- 브라우저 자동재생 정책 때문에 소리 재생은 사용자 버튼 입력 이후 시작하도록 설계했다.

## 실증 버전 확장

### 1. 방문 즉시 안부

- 첫 진입: "오늘 생활하면서 불편한 점이 있으셨어요?"
- 1차 후속 질문: "식사, 병원, 돈 걱정, 외로움 중 어떤 게 가장 가까우세요?"
- 2차 후속 질문: 상황별 복지자원 카드 추천

### 2. 예약형 안부

- 매일 1회 또는 주 2~3회 같은 시간에 안부 알림을 보낸다.
- 예시 문장:
  - "오늘 식사는 잘 챙겨 드셨어요?"
  - "요즘 혼자 계실 때 불편한 점은 없으셨어요?"
  - "병원이나 약 때문에 궁금한 점이 있으세요?"
- 웹앱에서는 PWA 알림으로, 운영 버전에서는 문자/알림톡/기관 담당자 연계 방식으로 확장한다.

### 3. 위험 신호 우선 대응

- "쓰러질까 봐 걱정돼요", "갑자기 아파요", "도와줄 사람이 없어요" 같은 표현은 응급·안전 우선 분기로 보낸다.
- 안내 순서:
  1. 지금 위험하면 119 또는 가까운 사람에게 연락하도록 안내
  2. 독거노인 응급안전안심서비스 등 공식 자원 추천
  3. 신청 대상, 지역, 신청 방법, 공식 URL 제공

## 음성 기술 선택

### 현재 MVP

- STT: 브라우저 Web Speech API
- TTS: 브라우저 `speechSynthesis`
- 장점: 비용 없이 즉시 배포 가능
- 한계: 브라우저/기기별 음성 품질과 지원 범위 차이

### 운영 버전

- STT: NAVER Cloud CLOVA Speech
- TTS: NAVER Cloud CLOVA Voice
- API 키는 프론트엔드에 넣지 않고 Vercel Serverless Function 또는 별도 백엔드에 보관한다.
- 국내 결제와 한국어 음성 품질을 고려하면 실증용 운영 스택으로 적합하다.

## 결과보고서 표현

> 찾아봇은 단순 검색형 인터페이스가 아니라 어르신에게 먼저 안부를 묻는 선제형 대화 구조로 설계하였다. MVP에서는 브라우저 내장 STT/TTS로 음성 질문과 음성 안내를 구현했으며, 실증 버전에서는 NAVER Cloud CLOVA Speech/Voice를 서버리스 API로 연동하여 한국어 음성 품질과 운영 안정성을 높이는 구조를 제안하였다.
