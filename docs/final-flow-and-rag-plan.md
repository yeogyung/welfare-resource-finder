# 최종 대화 플로우 및 RAG 확장 설계

## 최종 사용자 플로우

찾아봇의 홈 화면은 사용자를 "걱정" 하나로만 유도하지 않고, 필요와 욕구 두 축을 모두 열어 둔다.

- 필요 축: 끼니·생활, 외로움·말벗, 건강, 경제, 긴급 안전
- 욕구 축: 즐길 거리·모임, 새로운 배움

홈 첫 질문은 접속 또는 새로고침마다 랜덤으로 바뀐다. 입력창 예시 문구도 함께 바뀌어 사용자가 복지 용어를 모르더라도 자신의 말로 시작할 수 있게 한다.

## 후속 질문 구조

빠른 질문을 누르면 즉시 결과를 보여주지 않고, 상황을 한 번 더 좁히는 꼬리질문을 제공한다.

| 진입 질문 | 후속 질문 | 최종 추천 기준 |
|---|---|---|
| 끼니·생활이 걱정돼요 | 식사, 장보기·요리, 청소·집안일 중 선택 | 식사·생활지원, 방문형 돌봄, 복지관/센터형 자원 |
| 외롭고 말벗이 필요해요 | 거의 늘 혼자, 가끔 외로움, 사람들과 어울림 중 선택 | 안부 확인, 말벗, 사회참여, 여가 프로그램 |
| 건강이 걱정돼요 | 병원 이동, 의료비, 치매, 건강검진 중 선택 | 건강·의료, 보건소, 검진, 치료비 지원 |
| 생활비가 빠듯해요 | 매달 생활비, 일자리, 갑작스러운 위기 중 선택 | 경제·일자리, 기초연금, 긴급복지 |
| 즐길 거리·모임을 찾고 싶어요 | 경로당·모임, 취미·여가, 나들이·문화행사 중 선택 | 문화·여가, 사회참여, 복지관 프로그램 |
| 새로운 걸 배우고 싶어요 | 스마트폰, 키오스크·은행, 글·컴퓨터 중 선택 | 디지털·배움, 문해·정보화 교육 |
| 긴급하게 도움이 필요해요 | 현재 위험 여부 확인 | 119 우선 안내, 응급·안전 자원 |

## 현재 구현된 추천 방식

현재 배포 버전은 생성형 LLM 답변이 아니라 DB 기반 검색·추천 엔진으로 동작한다.

- `public/data/welfare-resources.json`: 정리된 복지자원 418건
- `public/js/chajabot-engine.js`: 생활형/여가·문화·배움/긴급 축 판별, 동의어 확장, 카테고리 가중치, 긴급 신호, 추천 근거 생성
- `api/recommend.js`: Vercel API에서 같은 추천 엔진을 실행해 검색 결과와 추천 근거를 반환
- 사용자 화면: 추천 카드에 대상, 지역, 기간, URL, 전화, 추천 근거 표시

## RAG 확장 방향

다음 단계에서 생성형 RAG를 붙일 때는 추천 엔진이 고른 Top-K 자원을 LLM context로 넘긴다.

1. 사용자 발화와 후속 질문 선택값을 합쳐 질의 생성
2. DB 추천 엔진으로 Top-K 근거 자원 검색
3. 검색된 자원의 `name`, `target`, `description`, `method`, `contact`, `url`만 context로 구성
4. LLM에는 "context 밖 정보는 생성하지 말 것"을 시스템 지시로 전달
5. 생성 답변, 검색 자원 ID, context를 저장
6. 교수님 피드백 기준에 맞춰 ROUGE, METEOR, BERTScore, Faithfulness, 환각 여부 평가

이 구조를 쓰면 사용자 화면은 계속 단순한 대화형 앱으로 유지하고, 모델 품질 평가는 내부 관리자 화면 또는 Colab 평가 노트북에서 분리해 수행할 수 있다.
