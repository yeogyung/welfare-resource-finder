# RAG 예비 평가 설계

## 반영한 교수 피드백

지도교수 피드백에 따라 단순 추천 결과 구현에 그치지 않고, LLM/RAG 답변의 성능과 신뢰성을 함께 평가하도록 설계했다.

- 단어 기반 유사도: ROUGE, METEOR
- 의미 기반 유사도: BERTScore
- 문맥 충실성: Faithfulness
- 환각율: DB에 없는 전화번호, URL, 신청 절차, 자격 조건 생성 여부

## 내부 평가 화면 구현

사용자용 프로토타입에는 평가 화면을 노출하지 않는다. 현재 정적 MVP에서는 `/admin.html`에서 외부 LLM 호출 없이 다음 예비 평가를 실행한다.

- 15개 실제 어르신 질문 시나리오
- 각 질문별 기대 키워드와 기대 카테고리
- Top-3 검색 결과에 기대 자원이 포함되는지 확인
- 추천 카드의 URL/신청방법/출처 존재 여부로 근거 충실성 예비 체크

## 평가 문항 예시

- 혼자 살다가 쓰러질까 봐 걱정돼요.
- 매달 생활비가 부족해요. 집은 있어요.
- 컴퓨터를 배우고 싶은데 어디로 가야 하나요?
- 병원비가 부담돼요.
- 치매가 걱정돼요. 어디에 상담하면 좋을까요?

## 향후 확장

운영 버전에서는 RAG 답변 생성 결과를 저장한 뒤 다음 항목을 자동 산출한다.

1. 정답 예시 답변과 생성 답변의 ROUGE/METEOR/BERTScore
2. 생성 답변 문장별 DB context 근거 매칭
3. DB에 없는 정보 생성 여부
4. 응급 질문에 대한 안전 안내 우선 노출 여부
5. 현장 실증 후 SUS/만족도/이해도와 결합 분석

## 결과보고서 표현

> 지도교수 피드백을 반영하여 사용자용 프로토타입과 내부 평가 기능을 분리하였다. 사용자 화면은 홈·대화·찜·설정 중심의 복지자원 발견 경험에 집중하도록 구성하고, RAG/검색 품질 평가는 별도 관리자 화면에서 15개 어르신 질문 시나리오를 기반으로 Top-3 적합도, 분류 적합도, 근거 충실성을 확인할 수 있도록 구현하였다.
